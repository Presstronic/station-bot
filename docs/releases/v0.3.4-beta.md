# Station Bot v0.3.4-beta

## Release Date

2026-06-05

## Summary

`v0.3.4-beta` delivers three bug fixes targeting reliability gaps discovered after `v0.3.3-beta.1`. Event reminders now route correctly to text channels for voice and stage events, include tier-based mention targeting, and guard against user-controlled mention escalation. Org membership detection no longer produces false negatives for members with hidden main orgs. Station timer voice notifications are reliable again after a regression in the voice-state detection path.

## Main Changes

- Event reminders for voice and stage events are routed to a derived text channel by naming convention (`{prefix}-*general`) instead of posting into in-voice text chat.
- Tier-based mention routing introduced: public voice events ping `@everyone`, org-tier events ping the configured org member role (falling back to `@here`), and division-tier events ping the matching guild role (falling back to `@here`).
- `/configure` event reminders channel setting replaced the free-text channel ID modal with a `ChannelSelectMenu` restricted to text channels.
- `@everyone` and `@here` in user-controlled event title and description fields are defanged to prevent mention escalation.
- Org membership check now cross-references the `visibility-V` CSS class on the RSI citizen profile page to detect hidden main org memberships that the orgs page alone misses.
- Station timer voice channel notification switched from `guild.members.fetch()` to `guild.voiceStates.fetch()` to reliably detect voice presence at timer expiry, with a `starterDisplayName` fallback when member fetch fails.

## Issues Addressed

- `#435` Event reminders were posting into voice channel in-call text surfaces instead of the correct text channel. Routing, mention targeting, and mention-escalation hardening all delivered.
- `#444` Org membership check returned false negatives for members whose main org is set to hidden. Fixed by cross-checking the profile page visibility signal.
- `#446` Station timer completion notification failed to detect whether the timer starter was still in a voice channel due to a regression in the member fetch path. Fixed with `voiceStates.fetch` and a display name fallback.
